# 04_SECURITY

This directory is intentionally reserved for future security architecture documentation.

Security-related rules and policies are currently documented across multiple files:

- **AGENTS.md** — Security Rules, Performance Rules, Logging Rules
- **API_ARCHITECTURE.md** — API security (JWT, RBAC, rate limiting)
- **AUTHENTICATION_MODULE.md** — Authentication security
- **DATABASE_ARCHITECTURE.md** — Database security (encryption, backups)
- **CODING_RULES.md** — Secure coding practices

A consolidated Security Architecture document should be created here when the security module is implemented (Phase 3+).
